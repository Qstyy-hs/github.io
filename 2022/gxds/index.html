数据库-关系代数语言
Post：2022-08-11 15:43:35 Tags：/ 数据库 / Visit：

关系代数是一种抽象的查询语言，用对关系的运算来表达查询，关系代数是关系数据库系统查询语言的理论基础。

一、集合运算

1.并∪

结果为二者元组之和去除重复行

2.交∩

结果为二者重复行

3.差➖

前者去除二者重复行

4.笛卡儿积 ✖

从数学角度理解，就是将集合A和集合B中所有有序对元素集合。在数据库系统中，表示得是对两个关系R和S进行操作，产生的关系中元组个数为两个关系中元组个数之积。

结果列数为二者属性列数之和,行数为二者元素数乘积。

二、关系运算

1.投影 π

关系R上的投影是从R中选择出若干属性列组成新的关系（竖直切割）

投影后不仅取消了原关系中的某些列，还有可能因为取消列后出现重复行，这些重复行也应该取消掉。

2.选择 σ

在关系R中选择满足给定条件的诸元组，根据某些条件对关系做水平切割

选择投影组合运算：πA( σF( R ) )

可以从特定元组中投影出特定列

从R关系中选出符合F条件的元组，在从这些元组内投影出特定的A列

3.连接

① 等值连接

θ称为比值符，也就是>、<、=号；

当θ两边的属性相等时，等值S连接成立；

② 自然连接 ⋈

R ⋈S = { t****r ⌒ t****s | t****r ∈ R ∧ t****s ∈ S ∧ t**r[A] = ts[A] }**

特殊的等值连接

结果列数为二者属性列数之和減去重复列,行数为二者冋名属性列其值相冋的结果元组。

笛卡尔积、选择、投影的组合表示可以与自然连接等价

扩展：自然连接和等值连接的联系和区别

⊙自然连接要求同名同域的属性组才可连接，而等值连接仅要求域相同；

⊙自然连接是去掉重复列的等值连接；

⊙自然连接一定是等值连接，而等值连接不一定是自然连接；

⊙自然连接后列的数量小于等于等值连接后列的数量。

4.除 ÷

除运算核心步骤：设R、S为关系，X、Y、Z为属性集

\1. 分出X、Y、Z三个属性集

\2. 列出S对Y属性集的投影

\3. 找出R中满足x值对应的象集包含了关系S对Y属性集的x值

运用技巧

1.首先看查询条件和结果中涉及到的属性列来自哪些表，将其归置到最少的表中进行查询。若涉及多个表则需要将表进行连接操作。

2.对于只涉及到选择、投影、连接的查询可用下列表达式表示：

π…****( σ…( R×S ) ) 或者 π…****( σ…( R S ) )

3.对于否定的操作，一般采用差操作表示

4.对于查询具有“全部”特征的操作，一般采用除法操作表示
